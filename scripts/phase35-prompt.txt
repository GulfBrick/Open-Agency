## BACKEND (src/agents/sales/)

Create src/agents/sales/lead.js — Sales Lead (Jordan):
- Extends BaseAgent from ../base-agent.js
- Model: claude-sonnet-4-5
- Methods:
  - qualifyLead(clientId, leadData) — scores lead, returns { score, tier: HOT|WARM|COLD, notes }
  - managePipeline(clientId) — returns full pipeline status from data/clients/[clientId]/pipeline.json
  - assignLead(leadId, agentId) — routes lead to closer/qualifier/followup
  - reportToNikita(clientId) — weekly sales summary
  - forecastRevenue(clientId) — pipeline value × close probability

Create src/agents/sales/closer.js — Closer (Zoe):
- Methods:
  - handleConversation(clientId, leadId, message) — responds to lead message in context
  - writeProposal(clientId, leadId, requirements) — delegates to proposal bot via registry
  - handleObjection(objection, context) — generates objection handling response
  - closeStrategy(clientId, leadId) — recommends close approach based on lead profile

Create src/agents/sales/qualifier.js — Lead Qualifier (Sam):
- Methods:
  - scoreLead(leadData) — BANT scoring (Budget, Authority, Need, Timeline), returns 0-100
  - categorizeLead(score) — HOT (70+) / WARM (40-69) / COLD (<40)
  - enrichLead(leadData) — adds context/questions to understand lead better
  - recommendAction(score, leadData) — next best action

Create src/agents/sales/followup.js — Follow-Up Bot (Maya):
- Methods:
  - scheduleFollowUp(clientId, leadId, lastContact, stage) — calculates next follow-up date
  - writeFollowUpMessage(clientId, leadId, context) — generates follow-up message
  - getDueFollowUps(clientId) — returns leads due for follow-up today
  - updateCadence(leadId, response) — adjusts follow-up schedule based on response

Create src/agents/sales/proposal.js — Proposal Writer (Leo):
- Methods:
  - writeProposal(clientId, leadId, requirements) — generates full proposal document
  - generateExecutiveSummary(requirements) — one-page summary
  - createPricing(requirements, pricingModel) — pricing table
  - writeProposalSection(section, context) — generates individual proposal sections
  - Saves proposals to data/clients/[clientId]/proposals/[leadId]-[timestamp].md

## CRM (src/core/crm.js)

Create src/core/crm.js — Simple CRM:
- Lead schema: { id, clientId, name, email, company, source, status: NEW|QUALIFIED|PROPOSAL|NEGOTIATION|WON|LOST, score, assignedTo, notes, createdAt, lastContact, nextFollowUp }
- Persists to data/clients/[clientId]/pipeline.json
- Methods:
  - addLead(clientId, leadData) — creates lead
  - updateLead(clientId, leadId, updates) — patches lead
  - getLead(clientId, leadId)
  - getPipeline(clientId, filter) — returns leads filtered by status/score
  - getStats(clientId) — { total, byStatus, totalValue, avgScore, conversionRate }
  - moveTo(clientId, leadId, status) — moves lead through pipeline stages
  - Singleton export
