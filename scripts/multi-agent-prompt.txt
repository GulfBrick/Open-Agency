## WHAT TO BUILD

### 1. src/core/agent-conversation.js — Agent Conversation Engine

This is the heart of multi-agent collaboration.

Each agent gets their OWN system prompt based on their SOUL.md persona file.
When Agent A needs to talk to Agent B, it creates a "conversation" where B actually responds IN CHARACTER using their own system prompt.

```
AgentConversation:
- conversationId
- participants: [agentId, agentId, ...]
- messages: [{ from, to, content, timestamp }]
- topic
- status: OPEN | WAITING_ON | RESOLVED
- clientId (optional)
- outcome (final decision/result)
```

Methods:
- createConversation(topic, initiator, participants, clientId) — starts a convo
- sendMessage(conversationId, fromAgentId, toAgentId, content) — one agent talks to another. ACTUALLY calls the recipient agent's brain with their own system prompt so they respond in character
- getConversation(conversationId) — full thread
- resolveConversation(conversationId, outcome) — marks done, logs outcome
- Persists to data/conversations/[conversationId].json
- Fires message bus events: CONVERSATION_MESSAGE, CONVERSATION_RESOLVED

### 3. src/core/workflow-engine.js — Workflow Engine

Orchestrates multi-step agent workflows. A workflow is a series of agent conversations that happen in sequence or parallel.

```
Workflow:
- workflowId
- name
- clientId
- steps: [{ stepId, type, agents, dependsOn, status, output }]
- status: PENDING | RUNNING | WAITING_APPROVAL | DONE | FAILED
- createdBy (agentId or 'harry')
- approvalRequired: boolean
- approvedBy (agentId or 'harry')
```

Built-in workflow templates:

DASHBOARD_REDESIGN workflow:
Step 1: Nova (creative-director) + Sage (architect) have a conversation about the dashboard redesign brief. Nova comes up with creative direction. Sage assesses technical feasibility. They come to an agreement.
Step 2: Nova and Sage bring their joint proposal to Nikita — she reviews it and decides whether to escalate to Harry for approval.
Step 3: [REQUIRES HARRY APPROVAL] Nikita presents the proposal to Harry and waits.
Step 4: Once approved, Nikita briefs Kai (dev-lead) to run the sprint. Kai assigns Luna (frontend) to build it.
Step 5: Luna builds. Atlas (code-review) reviews. Kai reports completion to Nikita.
Step 6: Nikita notifies Harry the job is done.

Methods:
- createWorkflow(template, clientId, brief) — creates a workflow from a template
- runWorkflow(workflowId) — executes steps in order, waits on dependencies
- approveWorkflow(workflowId, approvedBy) — Harry or Nikita approves a step
- getWorkflowStatus(workflowId) — full status with step details
- listWorkflows(clientId) — all workflows for a client
- Persists to data/workflows/[workflowId].json

## IMPORTANT RULES
- Every agent responds using their OWN persona from SOUL.md — they are NOT interchangeable
- Agent names must be used: "Nova says...", "Sage responds...", NOT "the creative director says..."
- Conversations are real — each agent calls the Anthropic API with their own system prompt
- Use claude-sonnet-4-5 for all worker agents (not opus — too expensive for multi-agent chains)
- Every conversation is logged to data/conversations/
- Every workflow is persisted to data/workflows/
- Load dotenv/config in every file that needs env vars
- Node.js ESM throughout

This is the most important file in the whole project. Build it carefully.
